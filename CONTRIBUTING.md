# Contributing Guide

## Purpose

This repository documents the Rays of Resilience ecosystem including ministry work, community projects, and supporting business platforms.

All contributions should maintain clear structure and documentation quality.

## Repository Structure

docs/overview  
Core documentation and planning

projects/ministry  
Ministry projects such as church development and orphanage operations

projects/business  
Business platforms supporting ministry funding

Root files  
README.md – Main entry point  
PROJECT_MAP.md – Ecosystem overview  
AGENTS.md – Instructions for AI systems

## Documentation Rules

Use clear titles and headings.

Each project must have its own markdown file.

Avoid duplicate documentation across multiple files.

Update the Project Map when adding major projects.

## Commit Messages

Use clear messages.

Examples:

Add Bugiri church documentation  
Update project roadmap  
Add Green Thumb Gardening Community documentation

## Goal

Maintain a clean, navigable documentation system that supports collaboration, planning, and future development.
